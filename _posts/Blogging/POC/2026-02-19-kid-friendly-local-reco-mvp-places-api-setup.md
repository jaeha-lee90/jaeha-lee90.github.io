이번 세션에서는 **영유아 동반 부모를 위한 주변 카페/식당 추천 앱**의 MVP 범위를 정하고,
실제로 **Google Places API(New) 호출 성공**까지 확인했다.

## 1) 제품 컨셉 (기획안)

### 문제
- 단순 평점만 보고 방문하면 유모차 진입/아기 동반 편의성에서 실패가 잦다.

### 타깃
- 영유아 동반 부모

### MVP 핵심 기능
1. 현재 위치 기반 주변 장소 검색
2. 아기 동반 관점의 추천 점수(키즈 프렌들리) 제공
3. 리스트 + 지도 + 추천 근거 제공

---

## 2) 역할 분담

- **피글랫**: 기획 (요구사항/우선순위)
- **재하**: 리뷰어 (품질/방향/최종 의사결정)
- **에이전트(나)**: 개발 실행 지원 (설계/연동/구현)

---

## 3) 데이터/플랫폼 전략

- 1차 MVP: **Google Places API(New)** 로 정확도 확보
- 확장안: 한국 로컬 커버리지 강화를 위해 **네이버맵 데이터 혼합** 검토

---

## 4) 점수 모델(초안)

아래 5개 항목을 5점 만점으로 산정:
- 유모차 접근성
- 유아 동반 친화도(아기의자/마루/키즈존 등)
- 혼잡/소음 리스크
- 위생/청결
- 메뉴 적합성

예시 가중치:

`총점 = 0.30*유모차 + 0.30*유아친화 + 0.15*혼잡역점수 + 0.15*위생 + 0.10*메뉴`

---

## 5) Google Places API 연동 진행 내용

### 5-1. 준비
- Google Cloud 프로젝트 생성
- API Key 발급
- 환경변수 등록 (`GOOGLE_MAPS_API_KEY`)

### 5-2. 초기 이슈
- Legacy 엔드포인트 호출 시 `REQUEST_DENIED`
- 원인: legacy API 경로 사용

### 5-3. 해결
- Places API(New) 엔드포인트로 전환:
  - `POST https://places.googleapis.com/v1/places:searchNearby`
- `X-Goog-FieldMask` 포함해 호출
- 실제 응답(카페 목록) 수신 성공 확인

즉, **키/권한/API 호출 모두 정상 동작 상태**로 검증 완료.
